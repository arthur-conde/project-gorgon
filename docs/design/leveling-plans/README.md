# Leveling Plans — UI design handoff (#228 PR-B)

**Tracked in:** #228 · foundation #369 (PR-A) · follow-ups #383 (XP-buff multiplier), #384 (plan share/import)

Visual & interaction specs for the #228 plan-aware leveling feature, generated by
Claude Design against the locked architecture. These are **design references for
XAML implementation, not shippable code** — the WPF build reuses the app's real
brushes / `WizardStepIndicator` / `MithrilDataGrid`; treat the HTML as
layout/interaction/state alignment only.

| File | Covers | Implements |
|---|---|---|
| `elrond-generate-plan-spec.html` | "Generate leveling plan" — skill+goal, embedded snapshot, preview, hand-off | **PR-B/B2** (Elrond module) |
| `plans-spec.html` | Plans manager (library) + walker (phase predicate, unlock callouts, sourcing, send-to-craft-list) | **PR-B/B1** (Celebrimbor module) |
| `colors_and_type.css` | Canonical design tokens (the spec HTML inlines its own copy; this is the reference) | both |

Each spec renders every named **state** as a separate annotated frame and carries
a `binds:` map per frame — those maps double as the B1/B2 ViewModel property
surface (e.g. `predicate.count ← CurrentEvaluation.OnHand`,
`pill.stale.visible ← IsInitialStateStaleAgainst(LiveCharacter)`,
`ing[i].source ↔ SourcingMode`). Build against the binds, not a pixel trace.

## Tokens (from the specs)

- Accent: **teal `#3DAFA0`** — reserved for the *primary action and progress
  only*. App bg `#1A1A1A`, surface `#1F1F1F`, border white @ ~13% alpha,
  three-step text ramp. Self-contained HTML (only an external Google-Fonts link
  for JetBrains Mono — degrades gracefully offline).

## Architecture reconciliation (apply when building)

The locked module split is **Elrond authors / generates the plan; Celebrimbor
receives, manages and walks it** (charter: Celebrimbor "*receives* a computed
plan … does not compute it"), with hand-off via a shared
`ISavedLevelingPlanImportTarget` (the #224 in-process pattern, generalised).

The mockup *chrome* frames authoring as a "Plans → Elrond" sub-area / points the
Celebrimbor empty-state at "the Picker tab" (`cta.openPicker → NavigateTo(Tab.Picker)`).
The controls / binds / states are correct and reusable **as-is**; only the
navigation framing bends to the locked split:

- B2's generate surface lives in the **Elrond module's** own UI, not a
  Celebrimbor "Plans" sub-tab.
- B1's Celebrimbor Plans empty-state copy/CTA points at **Elrond** (the advisor)
  as the plan source, not "the Picker tab".

## Scope

Only the two B1/B2 specs + token CSS are vendored. The full design-system
workspace (component gallery, brand previews, chat transcripts, asset kits) lives
in the Claude Design handoff and is intentionally **not** checked in — it's not
load-bearing for B1/B2 implementation.
