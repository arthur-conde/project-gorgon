notes

  1. Snapshot card is the embedded initial state. Teal left rule signals it travels into the plan record; not a status callout. ExportedAt renders as relative time, full timestamp in tooltip.
  2. Skill picker is focused on entry. Combobox bound to the player's known skills (keys of Skills). Typeahead filters; ↑↓ + ↵ commits.
  3. Current level dashed & read-only. Resolved from the snapshot once a skill is picked. "from snapshot" hint clarifies provenance, doesn't change.
  4. Preview is empty by design. No skeleton or stat zeros — communicates "no input yet" over "result is nothing". Recompute indicator appears in state 02.
  5. Primary action disabled. Stays disabled until SkillTarget.Skill ≠ null AND SkillTarget.GoalLevel > current. Keyboard shortcut still rendered to surface the affordance.

notes

  2. Goal accepts numeric input. Stepper + raw entry. Constrained 1–250; validation fires only on commit / blur. GoalLevel > CurrentLevel is the gating predicate for the action.
  3. Preview is the whole result surface. Four numeric facets in a tight row; the level range is the anchor (largest weight). "computed Nms ago" indicator confirms the recompute fired — important during fast iteration on the goal.
  4. Action is the only filled element. Teal reserved for primary + progress. Cancel stays ghost. Pressing Ctrl ↵ submits; success transitions to the Celebrimbor tab with the new LevelingPlan selected.
03 / NO CHARACTER

No active character — inputs disabled with inline explanation

The screen still renders so the user knows where they are, but the snapshot row is replaced with a warning that names the prerequisite. All form fields are dim and inert. The recovery action (import a snapshot) lives in the warning bar itself, not in the form.

04 / ALREADY AT GOAL

Goal ≤ current level — "Already at goal", action disabled

Smithing selected, goal entered as 10 (below the snapshotted current of 12). Not framed as an error — it's a calm "nothing to plan" state. Goal field carries a warm validation outline; the preview area carries the explanation. Action disables until goal exceeds current.
